## 💥 트러블 슈팅
1. 서브메뉴 반응형 작업을 위해 스크롤 생성 중, offsetLeft 값이 클릭할 때 잠깐 보이고 사라지는 이슈

   ⇒ **원인**: 메뉴 클릭 시 페이지가 이동되어 새로고침 되며 값이 사라짐. onclick function 내 명 
                 령 작성.

   ⇒ **해결 방법**: localStorage에 값을 저장하고, onclick function 밖에서 명령 작성.



2. 동적 데이터 렌더링 작업 중 화면에 'undefined'가 노출 되는 이슈

   ⇒ **원인**: 데이터의 구조가 달랐음에도 필터링 하지 않음.
               
   ⇒ **해결 방법**: 조건문을 이용하여 이슈가 발생한 'subtitle'값이 있을 때만 데이터를 저장하도록 
                      코드 수정.


3. 동적 데이터 렌더링 작업 중 작성해둔 스크롤 기반 컨텐츠 애니메이션이 2, 3번째 탭메뉴에서는 나타나지 않는 이슈

   ⇒ **원인**: 애니메이션이 페이지에 처음 진입했을 때 한 번만 실행됨.
               
   ⇒ **해결 방법**: 애니메이션을 함수 형태로 만들어 탭메뉴를 클릭할 때마다 실행되도록 코드 수정.




## 💭 프로젝트를 통해 배운점
이번 프로젝트를 통해 팀원 간의 소통이 얼마나 중요한지 실감했습니다.

처음 프로젝트를 시작할 때는 새로운 환경과 작업 방식에 대한 긴장감이 커서 어색함을 느꼈고, 그로 인해 소통이 부족했던 부분이 있었습니다.
각자 맡은 역할에만 집중하며 작업을 진행하다 보니 자연스럽게 대화할 기회가 적어졌고, 팀 프로젝트임에도 불구하고 반 이상은 개인적으로 진행하게 되었습니다.

그 결과, 수정사항이 예상보다 훨씬 많아져 이 이후로 저는 팀원들과 보다 적극적으로 대화를 시도했습니다. 팀원들 역시 더 많은 의견을 공유하며 협력적인 분위기가 형성되었습니다.

이렇게 서로의 의견을 존중하며 소통한 덕분에 최종적으로 만족스러운 결과물을 도출할 수 있었고, 팀원 간의 유대감도 자연스럽게 형성되었습니다.
앞으로는 프로젝트 초반부터 소통을 원활하게 하여 더 효율적이고 협력적인 작업 환경을 만들기 위해 노력할 것입니다.
